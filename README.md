# Minimal Blog Card

Este projeto é uma atividade de HTML e CSS para você colocar em prática o que já aprendeu.

## Objetivo

Reproduzir o layout do desafio **Minimal Blog Card** da devChallenges, seguindo o design proposto no link abaixo:

**[https://devchallenges.io/challenge/minimal-blog-card](https://devchallenges.io/challenge/minimal-blog-card)**

Use o link como referência visual: cores, tipografia, espaçamentos, tamanhos e estrutura do card devem ficar o mais próximo possível do layout apresentado.

## Como começar

### 1. Clonar o repositório

**Importante:** clone o repositório pelo Git. **Não baixe o projeto como arquivo ZIP.**

Clonar permite versionar seu trabalho com Git, acompanhar o histórico de alterações e facilita a revisão do código depois.

```bash
git clone https://github.com/SEU-USUARIO/minimal-blog-card.git
cd minimal-blog-card
```

> Substitua `SEU-USUARIO` pela URL do repositório que o mentor compartilhar com você.

### 2. Abrir o projeto

Abra a pasta do projeto no seu editor de código (VS Code, Cursor, etc.) e inicie pelo arquivo `index.html`.

Para visualizar no navegador, você pode:

- usar a extensão **Live Server** no VS Code, ou
- abrir o arquivo `index.html` diretamente no navegador.

### 3. Desenvolver a solução

O projeto já vem com uma estrutura inicial:

```
minimal-blog-card/
├── assets/
│   └── images/
│       └── placeholder-card.jpg   # imagem de referência para o card
├── index.html                     # estrutura HTML (comece por aqui)
├── style.css                      # estilos CSS
└── README.md
```

Sua tarefa é completar o HTML e o CSS para montar o card de blog conforme o layout do desafio.

## O que entregar

- Layout fiel ao design do [desafio na devChallenges](https://devchallenges.io/challenge/minimal-blog-card)
- Código organizado e legível
- Página responsiva (o desafio pede versões para desktop e mobile)

## Prazo estimado

**2 horas** para concluir e entregar.

Se precisar de mais tempo ou tiver dúvidas durante o desenvolvimento, converse comigo antes do prazo.

## Dicas

- Leia com atenção os requisitos e o preview do desafio no site da devChallenges antes de codar.
- Faça commits ao longo do desenvolvimento (`git add` + `git commit`) para registrar seu progresso.
- Teste em diferentes larguras de tela (desktop e mobile).
- Priorize HTML semântico e CSS limpo.

Boa sorte!
